# 项目管理流程设计

## 一、项目登记规范

### 新项目登记格式（群里@我时使用）
请按以下格式提供信息：

```
【新项目登记】
项目名称：XXX（必填）
项目类型：评估项目/咨询项目/可研项目/尽调项目/投标项目（必填）
项目单位：大陆/同致致信德/国德/中财宝信/江苏昌宏/其他（必填）
项目负责人：@某人 或 待定（必填，待定需后续补充）
最新情况：简要描述当前状态（必填）
合同金额：X万元（如已知）
项目概况：简要描述
目标来源：客户来源（如周/金/黄克诚等）
```

### 登记流程
1. 群里人员@我发送项目信息
2. 我检查是否重复（按项目名称+项目单位+项目概况匹配）
3. 确认必填项完整，如有缺失会提醒补充
4. 录入表格，记录登记人，自动填写登记时间（项目开始时间）
5. 同时写入操作日志表，记录登记人、登记时间、登记内容
6. 回复确认已登记

**必填项说明**：
| 字段 | 说明 | 格式 |
|------|------|------|
| 项目名称 | 必填 | 文本 |
| 项目类型 | 必填 | 五选一 |
| 项目单位 | 必填 | 六选一 |
| 项目负责人 | 必填 | @某人 或 "待定" |
| 最新情况 | 必填 | 文本，描述当前状态 |
| 登记时间 | 自动 | 取当前时间 |
| 登记人 | 自动 | 取发送消息的人 |

**注意**：项目负责人需在群里@某人，或填写"待定"。系统会自动将登记人记录为项目负责人（待定状态）。

## 二、阶段管理标准

### 合同阶段
| 状态 | 完成标志 | 正常时间 |
|------|----------|----------|
| 意向 | 客户有初步意向 | 1-3天 |
| 商谈中 | 正在沟通合同条款 | 3-7天 |
| 合同定稿 | 合同内容确定 | 1-2天 |
| 签订完成 | 双方签字盖章 | 1-3天 |
| 收到纸质 | 收到正式合同文本 | 1-2天 |

**超时提醒**：意向阶段超过7天、商谈中超过14天，提醒项目负责人跟进

### 现场阶段（多选）
| 状态 | 完成标志 | 正常时间 |
|------|----------|----------|
| 资料准备 | 列出所需资料清单 | 1-2天 |
| 现场勘查 | 完成实地勘查 | 1-3天 |
| 资料收集 | 收集到必要资料 | 3-7天 |
| 资料盖章 | 完成资料盖章 | 1-3天 |

**超时提醒**：各阶段超过正常时间2倍，提醒项目负责人

### 报告阶段
| 状态 | 完成标志 | 正常时间 |
|------|----------|----------|
| 测算结果 | 完成初步测算 | 3-5天 |
| 初步结果沟通 | 与客户沟通初步结果 | 1-3天 |
| 初步结果确定 | 客户确认初步结果 | 1-3天 |
| 报告初稿完成 | 完成报告初稿 | 3-5天 |
| 报告审核通过 | 内部审核通过 | 1-2天 |
| 报告出具提交 | 正式报告提交客户 | 1-2天 |

**超时提醒**：报告初稿完成后超过5天无进展，提醒项目负责人

### 收款阶段
| 状态 | 完成标志 | 正常时间 |
|------|----------|----------|
| 已开发票 | 开具发票 | 1-2天 |
| 款项收到 | 客户付款到账 | 7-30天 |
| 结算回公司 | 款项结算至公司 | 1-3天 |

**超时提醒**：已开发票超过30天未收到款项，提醒项目负责人催款

### 归档阶段
- 完成标志：资料整理归档
- 正常时间：报告提交后7天内
- 超时提醒：超过14天未归档，提醒项目负责人

## 三、提醒机制

### 每日检查（上午9:00）
检查以下内容：
1. 意向阶段超过7天的项目
2. 商谈中阶段超过14天的项目
3. 现场各阶段停滞超过2倍正常时间的项目
4. 报告初稿完成后超过5天无进展的项目
5. 已开发票超过30天未收款的项目
6. 报告提交后超过14天未归档的项目

### 提醒格式
```
【项目进展提醒】
以下项目需要关注：

1. 项目名称：XXX
   当前状态：XXX
   停滞时间：X天
   建议：XXX

...
```

## 四、表格更新规则

### 谁可以更新
- 群里任何人@我，提供更新信息
- 我负责在表格中执行更新
- 不接受直接编辑表格的请求

### 更新格式
```
【项目更新】
项目名称：XXX
更新内容：
- 合同状态：从XX改为XX
- 最新情况：XXX
- 报告状态：XX（如：报告初稿完成/报告出具提交）
- 报告编号：XXX（报告出具时填写）
- 出具时间：YYYY-MM-DD（报告出具时填写）
- 报告结论：XXX（报告出具时填写）
```

**报告出具信息**：当报告状态更新为"报告出具提交"时，可同时提供：
- 报告编号：报告的唯一编号
- 出具时间：报告正式出具的日期（YYYY-MM-DD）
- 报告结论：评估结果的具体数值或结论（如：年租金评估值为120万元）

**状态变更查询**：所有更新操作都会记录到本地日志，包括变更时间和变更内容。如需查询某个项目的操作历史（如"报告初稿什么时候完成的"），@我并提供项目名称即可检索。

## 六、常见问题

### 自动化流程报错
表格设置了"添加项目时提醒"自动化，可能出现字段类型不匹配错误。建议：
1. 进入表格 → 自动化流程 → "添加项目时提醒"
2. 检查流程中引用的字段类型是否与当前表格一致
3. 如不需要，可关闭或重新配置该自动化

**提示**：项目登记和提醒功能由我（AI助手）处理，不依赖表格自动化。
