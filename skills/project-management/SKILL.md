# 项目管理流程

用于管理 Feishu 多维表格中的项目全生命周期。

## 核心流程

### 1. 新项目登记

当用户在群里发送登记信息时：

1. **检查重复**：按「项目名称+项目单位+项目概况」匹配已有项目
2. **必填项校验**：确保以下字段完整
   - 项目名称（文本）
   - 项目类型（单选：评估/咨询/可研/尽调/投标）
   - 项目单位（单选：大陆/同致信德/国德/中财宝信/江苏昌宏/其他）
   - 项目负责人（@某人 或 "待定"）
   - 最新情况（文本，描述当前状态）
3. **自动填充**：
   - 项目开始时间 = 当前日期
   - 登记人 = 发送消息的用户
4. **录入表格**：写入 Feishu 多维表格
5. **记录日志**：写入本地操作日志（日期时间、操作人、操作类型、变更内容）
6. **回复确认**：告知用户登记成功

### 2. 项目更新

当用户要求更新项目时：

1. **定位项目**：按项目名称查询记录 ID
2. **执行更新**：修改指定字段
3. **记录日志**：记录变更前后的值、变更时间
4. **回复确认**

可更新字段包括：
- 合同状态（意向→商谈中→合同定稿→签订完成→收到纸质）
- 最新情况（文本）
- 报告状态（测算结果→...→报告出具提交）
- 报告编号、出具时间、报告结论（报告出具时）
- 收款状态、归档日期等

### 3. 操作日志

所有登记和更新操作必须记录到本地日志文件：

**日志路径**：`projects/project-management/operation-log.md`

**记录格式**：
| 日期时间 | 操作人 | 操作类型 | 项目名称 | 变更内容 |

**变更内容格式**：
- 登记：`新项目登记，类型：X，单位：X，金额：X...`
- 更新：`更新：字段名="新值"（由"旧值"变更），字段名="新值"...`

### 4. 每日检查提醒

定时任务（每天 9:00）检查以下大阶段超时情况（统一15天标准）：
- **合同阶段**：进入合同阶段（意向）后超过15天
- **现场阶段**：进入现场阶段后超过15天
- **报告阶段**：进入报告阶段（测算结果）后超过15天
- **收款阶段**：进入收款阶段（已开发票）后超过15天
- **归档阶段**：报告出具后超过15天未归档

只关注大阶段整体进度，小阶段内部不单独提醒。

阶段时间标准详见 [references/flow-design.md](references/flow-design.md)。

## 工具使用

### Feishu Bitable API

```javascript
// 查询项目
feishu_bitable_list_records({
  app_token: "MWT8bBCR3ai80QsLwjwc3ACvnJg",
  table_id: "tblmlBjhA1DDsZd4",
  page_size: 500
})

// 创建记录
feishu_bitable_create_record({
  app_token: "...",
  table_id: "...",
  fields: { ... }
})

// 更新记录
feishu_bitable_update_record({
  app_token: "...",
  table_id: "...",
  record_id: "...",
  fields: { ... }
})
```

### Cron 定时任务

```javascript
// 每日检查任务
cron({
  action: "add",
  job: {
    name: "项目进展检查提醒",
    schedule: { kind: "cron", expr: "0 9 * * *", tz: "Asia/Shanghai" },
    payload: { kind: "agentTurn", message: "检查项目表格..." },
    sessionTarget: "isolated",
    delivery: { mode: "announce", channel: "feishu", to: "群聊ID" }
  }
})
```

## 项目登记模板

群里@我时使用：

```
【新项目登记】
项目名称：XXX（必填）
项目类型：评估项目/咨询项目/可研项目/尽调项目/投标项目（必填）
项目单位：大陆/同致信德/国德/中财宝信/江苏昌宏/其他（必填）
项目负责人：@某人 或 待定（必填）
最新情况：简要描述当前状态（必填）
合同金额：X万元（如已知）
项目概况：简要描述
目标来源：客户来源（如周/金/黄克诚等）
```

## 日志查询

用户询问项目历史时：
1. 读取 `projects/project-management/operation-log.md`
2. 搜索项目名称
3. 按时间排序展示操作记录

## 注意事项

1. **所有操作必留痕**：登记、更新必须记录到本地日志
2. **必填项检查**：登记时缺少必填项必须提醒补充
3. **重复检查**：按项目名称+单位+概况匹配，避免重复登记
4. **项目负责人字段**：User 类型，需要飞书用户 ID，或先填"待定"
5. **自动化报错**：表格自带的自动化流程如有报错，建议用户手动关闭
