# 项目管理流程详细设计

## 一、项目登记规范

### 登记流程
1. 群里人员@AI发送项目信息
2. AI检查是否重复（按项目名称+项目单位+项目概况匹配）
3. 确认必填项完整，如有缺失会提醒补充
4. 录入表格，记录登记人，自动填写登记时间（项目开始时间）
5. 同时写入操作日志，记录登记人、登记时间、登记内容
6. 回复确认已登记

### 必填项
| 字段 | 说明 | 格式 |
|------|------|------|
| 项目名称 | 必填 | 文本 |
| 项目类型 | 必填 | 评估/咨询/可研/尽调/投标 |
| 项目单位 | 必填 | 大陆/同致信德/国德/中财宝信/江苏昌宏/其他 |
| 项目负责人 | 必填 | @某人 或 "待定" |
| 最新情况 | 必填 | 文本，描述当前状态 |
| 登记时间 | 自动 | 取当前日期 |
| 登记人 | 自动 | 取发送消息的用户 |

## 二、阶段管理标准

### 大阶段超时提醒（统一15天标准）

| 大阶段 | 进入标志 | 超时提醒 |
|--------|----------|----------|
| **合同** | 状态为"意向"或更后 | 进入合同阶段后超过15天 |
| **现场** | 任一现场选项被选中 | 进入现场阶段后超过15天 |
| **报告** | 状态为"测算结果"或更后 | 进入报告阶段后超过15天 |
| **收款** | 状态为"已开发票"或更后 | 进入收款阶段后超过15天 |
| **归档** | - | 报告出具后超过15天未归档 |

**说明**：只关注大阶段整体进度，小阶段（如资料准备、报告审核等）内部不单独提醒。

---

### 详细阶段说明（参考用，不用于超时提醒）

#### 合同阶段
| 状态 | 正常时间 |
|------|----------|
| 意向 | 1-3天 |
| 商谈中 | 3-7天 |
| 合同定稿 | 1-2天 |
| 签订完成 | 1-3天 |
| 收到纸质 | 1-2天 |

#### 现场阶段（可多选）
| 状态 | 正常时间 |
|------|----------|
| 资料准备 | 1-2天 |
| 现场勘查 | 1-3天 |
| 资料收集 | 3-7天 |
| 资料盖章 | 1-3天 |

#### 报告阶段
| 状态 | 正常时间 |
|------|----------|
| 测算结果 | 3-5天 |
| 初步结果沟通 | 1-3天 |
| 初步结果确定 | 1-3天 |
| 报告初稿完成 | 3-5天 |
| 报告审核通过 | 1-2天 |
| 报告出具提交 | 1-2天 |

#### 收款阶段
| 状态 | 正常时间 |
|------|----------|
| 已开发票 | 1-2天 |
| 款项收到 | 7-30天 |
| 结算回公司 | 1-3天 |

#### 归档阶段
- 正常时间：报告提交后7天内

## 三、提醒机制

### 每日检查（上午9:00）

检查以下大阶段超时情况：
1. **合同阶段**：进入合同阶段（意向）后超过15天的项目
2. **现场阶段**：进入现场阶段后超过15天的项目
3. **报告阶段**：进入报告阶段（测算结果）后超过15天的项目
4. **收款阶段**：进入收款阶段（已开发票）后超过15天的项目
5. **归档阶段**：报告出具后超过15天未归档的项目

### 提醒格式
```
【项目进展提醒】
以下项目需要关注：

1. 项目名称：XXX
   大阶段：合同/现场/报告/收款/归档
   当前状态：XXX
   已进入该阶段：X天（超过15天）
   建议：推进到下一阶段或确认状态

...
```

## 四、操作日志规范

### 日志路径
`projects/project-management/operation-log.md`

### 日志格式
| 日期时间 | 操作人 | 操作类型 | 项目名称 | 变更内容 |
|----------|--------|----------|----------|----------|

### 记录规则
- **登记**：记录完整项目信息
- **更新**：记录变更字段的前后值
- **日期时间格式**：YYYY-MM-DD HH:MM

### 示例
| 2026-03-03 11:57 | 唐林林 | 登记 | 扬子江金融城租金项目 | 新项目登记，类型：咨询项目，单位：大陆，金额：6.1万元 |
| 2026-03-03 14:51 | 唐林林 | 更新 | 扬子江金融城租金项目 | 更新：最新情况="报告已出"（由"报告初稿"变更） |

## 五、常见问题

### 自动化流程报错
表格设置了"添加项目时提醒"自动化，可能出现字段类型不匹配错误。建议：
1. 进入表格 → 自动化流程 → "添加项目时提醒"
2. 检查流程中引用的字段类型是否与当前表格一致
3. 如不需要，可关闭或重新配置该自动化

**提示**：项目登记和提醒功能由 AI 处理，不依赖表格自动化。

### 项目负责人字段
- 类型：User（飞书用户）
- 如已知负责人，使用 @某人
- 如暂未确定，填写"待定"
- 后续可通过更新操作补充
